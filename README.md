# Kanvis 个人主页 V1

这是康少 Kanvis 的静态个人主页，负责承接作者介绍、公开项目、课程说明和资源入口。

## 页面结构

- `/`：个人主页
- `/open/?tab=skill`：免费开放 Skill 生态
- `/open/?tab=workbench`：Kanvis Studio 说明页与本地运行边界

## 当前版本边界

- 公开页面不虚构收入、客户、案例、联系方式或课程付款链接。
- 服务区展示商务联系微信二维码，面向经营咨询、企业 AI 赋能和 AI 产品与品牌合作等明确需求。
- 首页与开放生态页的内测入口使用公开飞书表单收集真实问题、目标功能、当前工作流和微信号。
- 课程区先展示 `Kanvis AI 内容系统实操课` 的课程结构与 ¥980 价格，正式订阅地址确认后再接入。
- Kanvis Content OS 目前处于插件内测阶段，只作为课程和后续产品入口说明，不把尚未上线的网页版本伪装成可用功能。
- 工作台说明页只负责介绍：Skill 可从 Codex 直接进入 Kanvis Studio；Obsidian 集成通过 Kanvis Content OS 管理项目并进入同一个工作台，当前处于内测。本机完成项目编辑与 MP4 渲染，网页本身不运行本地工程。

## 本地预览

在本目录执行：

```powershell
python -m http.server 4173
```

然后打开 `http://localhost:4173/`。

## 上线

这是纯静态文件。公开源码放在 GitHub，正式站点优先部署到 EdgeOne Makers；不要上传项目执行库之外的私有素材、客户文件或商业系统目录。

### 正式站：EdgeOne Makers

1. 在 EdgeOne Makers 新建 Direct Upload 项目。
2. 上传本目录或当前发布 ZIP，选择 Global 加速区域。
3. 使用平台分配的 `*.edgeone.app` 免费 HTTPS 地址作为第一版官网。
4. 后续购买独立域名后，在同一个项目中绑定，不需要重做页面。

这条链路更适合同时面向中国大陆与海外用户。EdgeOne 地址是免费的平台子域名，不等于免费拥有独立顶级域名。

### GitHub：开源与版本管理

GitHub 仓库负责公开源码、版本记录、Issue 和后续自动部署。`github.io` 可以用于技术预览，但不作为商业主页主域名，也不承接付款等敏感交易。

推送约定：

```powershell
git push origin main
```

正式站后续可从 EdgeOne Makers 连接 GitHub 仓库，推送 `main` 后自动更新。

### 公开表达规则

- 对外页面只回答：适合谁、解决什么、如何开始。
- 付款状态、申请筛选、社群分层、客户备注和回复边界属于内部运营信息，不写入公开页面。
- 对外文案始终站在访问者视角，让用户感到被理解和被邀请，而不是被后台规则管理。
- 后续可增加公众号、微信小店或课程下单入口，但必须先确认公开身份与使用场景。

正式课程订阅链接确认后，只需要修改 `index.html` 中的课程 CTA，并重新上传整个目录。
